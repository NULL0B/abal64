accept document parameters

This visual form is presented to allow the user the chance to defin e various document specific parameters prior to the generation of the HTML user guide.

TitleHeading
The TITLE edit field will be used in the TITLE of the Document HEAD section and in the initial HEADING of the document BODY.

ScreenStyle
The SCREEN STYLE edit field allows the name of an external style sh eet to be specified.

This style sheet will be used for Screen fo rmating of the document or document suite and should contain style instrections for TABLE,TH,TD and HEADING items.

PrintStyle
The PRINT STYLE ezdit field allows an external style sheet url to b e specified for document printing operations.

Logo
The LOGO edit field allows an url for a logo file name to be specif ied. The image logo will be displayed to the right of the title tex t at the top of all relevant pages.

FooterCopyright
The FOOTER edit field allows copyright or other information to be included at the ens of the document to be specified.

Separate Page Generation
This check box controls the way in which the document is generated. If this control is checked then the document will be generated as a collection of HTML pages, one per subcomponent, the master document comprising simply a header section and Table of Contents. Otherwise the master document will be generated to contain documentation for all components in one Html File.

Force Language
This check control, when activated, indicates that the selected national language is to be used for the production of all subsequent html documents. Otherwise the language code of each individual subcomponent will be respected.

NationalLanguage
This select control allows the national language to be used by the html document production mechanisms to be specified. This value will be ignored if the Force Language control is not active.

Parts Listing
This check box when selected directs the HTML production mechanisms to generate a table of component providing detail for each component. When unchecked the document will include only the user guide documentation texts and images.

Accept
As a result of clicking on the ACCEPT button the user guide documen t of the current application model will be generated using the spe cified parameters.

Cancel
USer guide document generation may be cancelled at this stage by clicking on the CANCEL button of this visual form.
