Sing Version 1.0a

visual debug

This document describes the new visual user interface mechanisms that have been added to the standard ABAL Debug Utility exadb.

The new user interface has been entiry redesigned using the ABAL Visual Design tools.

The environment variable VEXADB must be set indicating the name of a valid configuration file or LDAP entry, depending on the state of the ABALHOST environment variable.

If the VEXADB variable is not set then exadb will operate in standard text mode, either enhanced X/11 text mode using the GIGO variable, or the standard CICO default text mode mechanisms.

  1. Program Source
    This tab page displays the current position in the current source of the program under debug control using the new dynamic library form of the abal editor aed.

    All standard editor positioning keys and mouse events are accepted, other than page up and page down which will be interpreted by the visual user interface as debug window tab page commands.

    • exit.gif
      If this push button is activated the current debug context will be abandoned and control will be returned to the calling process or environment.

    • step.gif
      By pressing this button the program execution will step to the next logical new line.

    • goto.gif
      This button may be pressed to cause program execution to be continued up to the line indicated by the current source editor window cursor position. The source editor window cursor may be positioned by simply clicking on the desired line in the program source window.

    • trap.gif
      This button may be pressed to add break points at each occurance of the source line indicated by the current source editor window cursor position. The source editor window cursor may be positioned by simply clicking on the desired line in the program source window.

    • clear.gif
      This button may be pressed to remove all break points defined for the line indicated by the current source editor window cursor position. The source editor window cursor may be positioned by simply clicking on the desired line in the program source window.

    • run.gif
      If this push button is activated then program execution will continue until one of the various break conditions is encountered or program execution is terminated due to an instruction STOP.

    • view.gif
      This button may be pressed to cause the variable name under the current source editor window cursor position to be displayed in the status window to this effect at the bottom of the screen. The right button of the mouse will perform the same effect over a variable name in the source window. The source editor window cursor may be positioned by simply clicking on the desired line in the program source window.

  2. Program Output
    This tab page presents the current image of the visual application output screen.

    It is captured during program instruction execution by the visual buffering mechanisms of the visual library and redisplayed when this page is active.

    • Debug Control
      This tab page presents the various debug controls allowing break point management, variable, file, code section and source management functions to name but a few.

      • ProcName
        This select control presents the list of all procedures in the current active tcode program.

        As debug control is passed between overlays the corresponding list of procedures will be loaded into this list.

        The list is opened by a simple click on the button of the edit zone.

        As for all select controls, of the visual library, list scrolling may be performed by standard keyboard and mouse operations.

      • SegmName
        This select control presents the list of all segments in the current active tcode program.

        As debug control is passed between overlays the corresponding list of segments will be loaded into this list.

        The list is opened by a simple click on the button of the edit zone.

        As for all select controls, of the visual library, list scrolling may be performed by standard keyboard and mouse operations.

      • GlobNames
        his select control presents the list of all global variables in the current active tcode program.

        As debug control is passed between overlays the corresponding list of global variables will be loaded into this list.

        The list is opened by a simple click on the button of the edit zone.

        As for all select controls, of the visual library, list scrolling may be performed by standard keyboard and mouse operations.

      • LocalNames
        This select control presents the list of all local variables of the current active tcode procedure or segment of the current program.

        As debug control is passed between different procedures and segments the corresponding list of local variable will be made available.

        The list is opened by a simple click on the button of the edit zone.

        As for all select controls, of the visual library, list scrolling may be performed by standard keyboard and mouse operations.

      • AssignList
        This select control presents the list of assign table entries for the current tcode execution context.

        Information will be added to this list as files are assigned by the instruction ASSIGN.

        The list is opened by a simple click on the button of the edit zone.

        As for all select controls, of the visual library, list scrolling may be performed by standard keyboard and mouse operations.

      • RegisterList
        This select control displays the current contents of all registers of the ABAL virtual machine.

        The values of the various registers will be changed in response to the execution of the various tcode instructions of the tcode context.

        The list is opened by a simple click on the button of the edit zone.

        As for all select controls, of the visual library, list scrolling may be performed by standard keyboard and mouse operations.

      • AttachList
        This select control presents the list of program overlays attached to the current tcode execution context.

        Items will be added to this list as a result of the instruction ATTACH and removed from the list in response to the instruction DETACH.

        The list is opened by a simple click on the button of the edit zone.

        As for all select controls, of the visual library, list scrolling may be performed by standard keyboard and mouse operations.

      • StackList
        This select control displays the stack trace of calls leading to the current execution point of the current tcode execution context.

        Entries will be added for each level of nesting of GOSUB, LDGO.SEG and CALL as control is passed between overlays,procedures and overlays .

        The list is opened by a simple click on the button of the edit zone.

        As for all select controls, of the visual library, list scrolling may be performed by standard keyboard and mouse operations.

      • LibraryList
        This select control presents the list of libraries declared by the current active tcode program.

        As debug control is passed between overlays the corresponding list of libraries will be loaded into this list.

        The list is opened by a simple click on the button of the edit zone.

        As for all select controls, of the visual library, list scrolling may be performed by standard keyboard and mouse operations.

      • ObserveList
        This select control presents the list of active variable observation requests.

        Itsm will be added and removed from list list in response to the various command and controls events to this effect.

        The list is opened by a simple click on the button of the edit zone.

        As for all select controls, of the visual library, list scrolling may be performed by standard keyboard and mouse operations.

      • -
        By clicking on this button the current selected value of the active observations list will be disactivated, deleted and removed from the list.

      • BreakList
        This select control presents the list of all active break points in the current tcode debug context.

        Items will be added to this list in response to the various set break point commands and controls.

        Items will be removed from this list in response to the various clear break point commands and controls.

        The list is opened by a simple click on the button of the edit zone.

        As for all select controls, of the visual library, list scrolling may be performed by standard keyboard and mouse operations.

      • -
        By clicking on this button the current selected value of the active break points list will be disactivated, deleted and removed from the list.

      • WatchList
        This select control presents the list of active memory watch requests.

        Items will be added and removed from this list in response to the various command and control events provided to this effect.

        The list is opened by a simple click on the button of the edit zone.

        As for all select controls, of the visual library, list scrolling may be performed by standard keyboard and mouse operations.

      • -
        By clicking on this button the current selected value of the active watch points list will be disactivated, deleted and removed from the list.

      • SourceList
        This select control presents the list of all source files which contributed to the construction of the current active tcode program.

        As debug control is passed between overlays the corresponding list of files will be loaded into this list.

        The list is opened by a simple click on the button of the edit zone.

        As for all select controls, of the visual library, list scrolling may be performed by standard keyboard and mouse operations.

      • >
        By clicking on this button the current selected value of the source files list will be opened and displayed in the program source window.

        The visual focus will then be transfered to the program source tab page.

      • Syntax
        This edit field is used for output of results from variable display control events and for the manual input of standard exadb command expressions.

      • AED_NORMAL
        This select control allows selection of the colour to be used by the internal ABAL text editor to represent normal text, user defined naming tokens for variables, procedures and functions.

      • AED_QUOTE
        This select control defines the colour to be used to represent quoted string literal values.

      • AED_KEYWORD
        This select control defines the colour to be used to represent language keywords.

        The internal ABAL text editor may operate with syntaxical colouration for the following source languages :

        • ABAL and ABAL++
        • C and C++
        • HTML

      • AED_COMMENT
        This select control defines the value to be used to represent language dependant commentary blocks.

      • AED_PUNCTUATE
        This select control defines the colourto be used to represent standard punctuation characters,arithmetical and comparision operators.

      • Multiple
        By clicking on this button, a multiple hit break point will be defined for the procedure currently selected by the procedure selection control.

        This is equivalent to the standard exadb command syntax APprocname

      • One Shot
        By clicking on this button, a one shot break point will be defined for the procedure currently selected by the procedure selection control.

        This is equivalent to the standard exadb command syntax BPprocname

      • Multiple
        By clicking on this button, a multiple hit break point will be defined for the segment currently selected by the segment selection control.

        This is equivalent to the standard exadb command syntax ASsegment

      • One Shot
        By clicking on this button, a one shot break point will be defined for the segment currently selected by the segment selection control.

        This is equivalent to the standard exadb command syntax BSsegment

      • Multiple
        By clicking on this button, a multiple hit variable watch request will be added to the watch list for the variable currently selected by the global variable selection control.

        This is equivalent to the standard exadb command syntax AVvariable

      • One Shot
        By clicking on this button, a one shot variable watch request will be added to the watch list for the variable currently selected by the global variable selection control.

        This is equivalent to the standard exadb command syntax BVvariable

      • Observe
        By clicking on this button, a variable observation request will be added to the variable observation list for the variable currently selected by the global variable selection control.

        This is equivalent to the standard exadb command syntax OAvariable

      • Inspect
        By clicking on this button the current value of the variable currently selected by the global variable selection control will be displayed in the command and display zone of the debug control page.

        This is equivalent to the standard exadb command syntax Dvariable or DHvariable depending on the state of the debug control HexaDecimal.

      • Observe
        By clicking on this button, a variable observation request will be added to the variable observation list for the variable currently selected by the local variable selection control.

        This is equivalent to the standard exadb command syntax OAvariable

      • Inspect
        By clicking on this button the current value of the variable currently selected by the local variable selection control will be displayed in the command and display zone of the debug control page.

        This is equivalent to the standard exadb command syntax Dvariable or DHvariable depending on the state of the debug control HexaDecimal.

      • BreakKey
        This control switch controls the state of the execution interrupt mechanism. When active, execution may be interrupted by pressing the escape key. Otherwise execution will continue un-interrubted till break point or STOP.

      • AsmList
        This control switch controls the state of the execution interrupt mechanism. When active, execution may be interrupted by pressing the escape key. Otherwise execution will continue un-interrubted till break point or STOP.

      • HexaView
        The state of this switch controls the way in which variable inspection operations are performed. When hexa is active the variable value will be displayed as a string of hex bytes, otherwise the value will be displayed using its standard print format.

    • Variables
      This tab page presents the various debug controls allowing break point management, variable, file, code section and source management functions to name but a few.

      • vListe
        This table display the list of all global variables in a convenient form for viewing. A variable value may be sent tothe memory dump mechanism by clicking on the corresponding line of this table. Virtual file variables for which an assign has not been performed will be signaled by the standard ERROR 116 . Pointer variables which are invalid are indicated by the standard ERROR 118.
