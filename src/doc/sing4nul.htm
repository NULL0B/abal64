Introduction

Ce document est conçu pour aider les nouveaux venus à démarrer avec l'outil de design graphique abal3 connu sous le nom de Sing.

Le lecteur sera guidé pas à pas, à travers différentes étapes et cycles, et impliqué dans le développement d'une application simple mais cependant complète, en utilisant cet outil.

Les différentes opérations présentées dans ce document sont également disponibles sous forme de scripts d'animation qui peuvent être visualisés par l'outil de design. Cela permet une confirmation visuelle des descriptions textuelles en cas de difficulté rencontrée dans la compréhension des manipulations ou techniques en cours .

Installation

L'installation de Sing se produit pendant l'installation d'un package auto-installable de abal 3 de type abal3.1a.x.xx.

Ce package devrait être exécuté en précisant le chemin d'accès complet.

Depuis la version abal3.1a.0.15, l'installation exécute toutes les opérations nécessaires, aussi bien sous Useit 1.3 que sous Useit 1.4.

Les packages antérieurs à cette version ne peuvent s'installer que sous Useit 1.3

La structure de répertoire suivante est créée par la procédure d'installation.

/home
/abal3
/bin
/lib
/class
/include
/fonts
/images
/examples
/doc
/en
/fr

Le contenu du répertoire /home/abal3/bin est copié vers le répertoire /opt/prologue/bin.

Le contenu du répertoire /home/abal3/lib est copié vers le répertoire/lib.

Les fichiers de configuration ldap /home/abal3/bin/abal3.schema et /home/abal3/bin/abal3.access sont copiés vers le répertoire ldap /etc/openldap/schema.

Suivant la version du sytème, des fichiers de configuration ldap seront modifiés

Useit 1.3b
Le fichier /etc/openldap/slapd.conf est modifié pour inclure le fichier /etc/openldap/schema/abal3.schema. Ce fichier fournit la description des classes ldap requises par Sing et Visual pour la description de leurs objets de configuration.

Useit 1.4
Les fichiers /etc/openldap/schema/language.schema et /etc/openldap/schema/language.access sont modifiés pour inclure respectivement les fichiers /etc/openldap/schema/abal3.schema et /etc/openldap/schema/abal3.access .

le fichier /etc/profile.prologue est modifié pour définir les variables d'environnement requises par Sing et Visual pour une exécution correcte

ABALHOST
Cette variable est fixée à l'adresse IP locale de la machine hôte (host)

Elle est utilisée par Sing et Visual pour l'identification de leur serveur ldap devant être utilisé pour la récupération des informations de configuration.

VEXADB
Cette variable définit le nom de l'ensemble de configuration qu'utilise l'utilitaire de débogage visuel abal 3.

SINGXML
Cette variable définit le schéma XML qui doit être utilisé par Sing.

Quand elle est présente, cette variable provoque la génération et le chargement d'une ressource XML à activer par défaut.

A la suite du processus d'installation, un nouveau terminal est démarré pour s'assurer que les diverses variables d'environnement sont correctement positionnées.

Configuration

Une fois l'installation terminée Sing ne nécessite pas de configuration additionnelle pour être prêt à l'utilisation.

Si la configuration ldap est disponible, Sing peut être lancé de n'importe quel répertoire , la totalité des informations de configuration sont prises depuis le serveur de répertoire ldap défini par la variable d'environnement ABALHOST.

Dans des situations où la configuration ldap n'est pas possible ou non souhaitée, Sing devra être lancé avec un fichier de configuration valide présent dans le répertoire de travail.

Pour plus d'information au sujet des différents élements de configuration pour Sing et Visual vous pouvez consulter le document Configuration de Sing.

Prise en main

La suite de ce document est consacré à la présentation d'un exemple concret de l'utilisation de Sing pour le développement d'un exemple d'application réduit mais cependant exhaustif.  

En premier lieu, nous créons un répertoire de travail dans le cadre de cette présentation dans lequel tous les aspects du projet seront enregistrés...

mkdir /home/walkthrough

... nous nous positionnons ensuite dans ce nouveau répertoire comme le répertoire de travail courant ...

cd /home/walkthrough

... et nous démarrons Sing, depuis la ligne de commande, en indiquant le nom de notre fichier de projet, celui dans lequel nous voulons travailler, appelé dans ce cas myproject.

sing myproject

A ce point, l'écran devrait approximativement ressembler à l'image suivante, les couleurs pouvant différer, mais les divers éléments étant identiques.

Fig 1. Ecran de démarrage Sing

Ceci est référencé comme la vue d'Editeur de Projet et affichera en temps voulu tous les items attachés à ce projet.

Votre attention est attirée , à ce point, par les diverses zones de l'écran initial de démarrage

Vous pourrez remarquer

  1. La barre de menu en haut de l'écran, comprenant plusieurs options de menu et plusieurs boutons poussoirs.

  2. La barre de statut en bas de l'écran comprenant trois zones d'affichage

  3. Le bureau ou zone de travail dans le milieu de l'écran qui peut être noir et complètement vide.

Ces trois éléments sont présents au cours de l'éxécution de Sing, leur contenu pouvant varier en fonction de l'éditeur, occupant couramment l'espace de travail central.

A gauche et à droite du menu supérieur, vous pouvez noter deux boutons poussoirs en forme de flèche.

Celui sur la gauche de la barre de menu contrôle l'affichage de la barre d'outils contextuelle ou boîte qui peut apparaître ou disparaître en cliquant sur ce bouton..

Le bouton en forme de flèche à droite de l'écran contrôle de la même manière l'affichage de la barre de palette ou boîte qui peut apparaître ou disparaître en cliquant sur de bouton.

Esssayez maintenant ces deux opérations pour vous familiariser avec leur comportement.

Chacune des options de menu textuelles s'ouvre pour présenter une boîte de menu des choix subordonnés par un simple clic souris, ou plus précisément un événement de bouton "relaché", sur le texte correspondant.

Ces boîtes de menu peuvent être changés par un clic ultérieur sur une autre option de menu principal ou peuvent disparaître par un clic souris sur une autre partie de l'écran.

les choix subordonnés présentés par les boîtes du menu principal peuvent être activés par un clic ou un événement "relaché" d'un bouton de la souris qui transférera le contrôle à la fonction sous-jacente correspondante.

En examinant plus dans le détail la barre de menu, on peut noter que les options de menu textuelles présentent un caractère déclencheur. La façon dont cela est représenté dépend de la configuration et peut aussi bien être

  1. souligné
  2. gras
  3. d'une couleur différente

Ce déclencheur peut être utilisé en conjonction avec la touche Alt du clavier comme un raccourci pour l'activation de l'option de menu correspondante.

A droite de l'écran, mais à gauche du bouton de contrôle de la barre de palette, il faut noter un ensemble de cinq boutons d'activation d'utilitaires.

Le comportement de ces boutons est contextuel et dépend de la nature de la cible à laquelle ils seront appliqués mais ils peuvent à peu près être considérés comme représentant les fonctions utilitaires de base suivantes

  • Execution en mode Debug
  • Execution Normale
  • Configuration de l'Editeur
  • Traduction
  • Lien Objet

Chacun de ces cinq boutons peut être actionné par un clic ou un événement "relaché" de la souris.

Quand le bouton gauche de la souris est utilisé, la fonction soulignée est actionnée directement.

Le bouton droit de la souris néanmoins active la boîte de dialogue correspondante pour une éventuelle modification et confirmation des paramètres associés à cette opération.

Essayez maintenant de cliquer sur ces boutons d'utilitaires avec le bouton droit de la souris pour vous familiariser avec leur comportement.

La barre de status en bas de l'écran présente trois zones de message.

Affiché le plus à droite, vous pouvez noter le message Xml parse failure.

C'est normal tant que le fichier passé comme paramètre à la ligne de commande pour l'invocation de Sing n'existe pas encore (pas encore enregistré).

La zone de message la plus à gauche dans la ligne de status montre la position courante de la vue du projet.

La zone de travail centrale, couramment utilisée par l'éditeur de projet, est un espace tridimensionnel qui peut être déplacé le long des trois axes par divers événements souris et actions des contrôles de barre d'outils.

Pour plus d'information sur cet élément , vous pouvez consulter le document Editeur de Projet Sing correspondant.

  Maintenant, nous pouvons ajouter notre premier composant dans le projet. Cela peut être un fichier source abal pour montrer la façon dont les fichiers de programmes et d'inclusion (include) peuvent être ajoutés au projet. Activez l'option Projet du menu principal, par une séquence de raccourci clavier Alt P ou en cliquant avec la souris sur le texte de l'option dans le menu .

Activez ensuite l'option de menu subordonnée Ajout d'un Source qui charge la boîte de dialogue Propriétés de l'objet 3D.

Vous devez à ce niveau entrer le nom du fichier source et pour cela, comme premier exemple, taper program.as dans le champ de saisie Identité puis cliquer sur le bouton Valider.

Vous retournez à la vue de l'éditeur de projet où un petit rectangle blanc affichant le texte program.as devient visible.

Nous ouvrons maintenant un fichier texte vide pour être capables de taper les instructions de notre premier programme source.

Pour faire cela
activez la séquence de menu Vue . Visual
ou,
en utilisant la souris, déplacez la souris au dessus du rectangle blanc, appuyez sur le bouton gauche , cliquez sur le bouton droit, puis relachez le bouton gauche.

Dans les deux cas le fichier texte est ouvert, dans l' Editeur Texte et le contenu vide du fichier, program.as, est affiché dans la zone de travail.

Simultanément, vous pouvez noter que l'option du menu principal Projet a disparu pour être remplacé par la nouvelle option de menu Editeur.

L'éditeur texte interne utilisé par Sing est l'éditeur texte standard abal AED mais mis sous forme d'une bibliothèque dynamique pour faciliter l'intégration.

Toutes les fonctions de l'éditeur standard abal sont utilisables.

Vous pouvez noter les différentes cases de statut et le compteur de lignes qui apparaît également dans la barre de menu en haut de l'écran, à gauche des boutons d'utilitaires. Ceux-ci sont présents uniquement pour information donc n'essayez pas de cliquer dessus.

La première case de status montre le statut INSERT de l'éditeur, qui peut être basculé par la touche INS ou la combinaison standard de touches Ctrl O, il est toujours actif quand l'éditeur démarre.

La seconde case de statut montre le statut SELECTION de l'éditeur, et peut être basculé par la combinaison de touches Ctrl W ou par les techniques de sélection du bouton droit de la souris, il est toujours inactif quand l'éditeur démarre.

A ce point, vous pouvez taper les instructions du programmme élémentaire suivant 

Program " "
Segment 0
Pause "Abal"
Eseg 0
End

Quand vous avez terminé de taper les instructions ci-dessus, activez la séquence Fichier.Sauver dans le menu principal, pour enregistrer le texte de votre fichier, suivi par la séquence Fichier.Quitter dans le menu principal pour retourner à l'Editeur de projet.  

Nous pouvons maintenant traduire notre fichier de programme source en activant le bouton d'utilitaire Traduction de la barre de boutons d'utilitaires en haut de l'écran.  

Une fois la traduction terminée, ce programme élémentaire peut être exécuté par le bouton d'utilitaire Exécution Normale ou par le bouton Exécution en mode Debug dans la barre de boutons d'utilitaires en haut de l'écran.

Vous êtes encouragés à répéter ces opérations autant que nécessaire pour vous familiariser avec les différentes étapes. Ce serait également une bonne chose, avant de continuer, d'enregistrer l'état actuel de votre projet dans un fichier projet.

pour cela, activez l'option Fichier.Sauver sous du menu principal pour accéder à la boîte de dialogue enregistrement du modèle de projet.

Vous devez vous assurer que le fichier projet a le nom requis; il peut être changé si vous le souhaitez. Cliquez ensuite sur le bouton Valider pour confirmer cette opération.

Le nom de fichier du projet doit avoir une extension de fichier .xml pour permettre à Sing d'enregitrer la description de projet en utilisant un format XML RESSOURCE.

Vous pouvez ensuite sortir de Sing et revenir à la ligne de commande du système hôte en activant la séquence Fichier.Quitter du menu. Si vous n'avez pas encore enregistré votre projet, la boîte de dialogue Validation Sortie s'affiche, vous permettant d'enregistrer, d'abandonner la modification ou annuler l'opération de sortie.

Ce serait maintenant une bonne chose de réexécuter Sing avec la syntaxe d'origine de la ligne de commmande

sing myproject

pour s'assurer que le projet enregistré peut être correctement rechargé.

Dans ce cas, le message XML parser failure ne sera plus visible dans la zone droite de la ligne de message dans la barre de statut.

 

Avant de passer à la prochaine section, il faut remarquer que cette technique peut être appliquée à un ensemble de fichiers sources existants incluant une application abal existante.

Un ensemble de fichiers sources peut être attaché à un projet tout aussi facilement par l'activation de la séquence Projet.Ajout d'un source dans le menu. Cette fois, une spécification joker (wildcard) est fournie, représentant l'ensemble des sources à attacher.

par exemple

/home/users/monappli/*.as

ou

/home/abal3/include/*.as

Tous les fichiers correspondant aux critères de recherche sont par conséquent attachés au projet, et représentés par des rectangles blancs contenant le nom de fichier.

La même chose peut être faite par la séquence de menu Projet.Ajout Image, permettant d'attacher un ensemble d'images à un projet.

 

Nous allons maintenant continuer notre tour guidé de Sing et procéder à la création d'un autre type de composant de projet : notre premier formulaire visuel.

Vous devez activer la séquence de menu Projet.Ajout formulaire, et préciser le nom de fichier du formulaire visuel devant être créé, dans le cas de notre exemple, tapez example, et cliquez sur le bouton Valider pour valider l'opération et pour obtenir l'affichage de la boîte de dialogue Chargement de Formulaire en indiquant le nom du formulaire à charger. L'extension XML est ajoutée à notre nom de fichier souche example, étant donné que tous les formulaires visuels sont stockés et chargés comme des fichiers RESSOURCE XML. En activant le bouton Valider vous revenez à la vue d'éditeur de projet comme précédemment et vous pourrez remarquer

  1. un rectangle gris qui a été ajouté au centre de la vue projet.
  2. Le message xml parse failure montre que le formulaire n'existe pas.

Ce nouveau composant formulaire peut maintenant être ouvert, comme pour le fichier source précédent, par la combinaison de clics souris gauche et droit au dessus du rectangle du composant de projet ou par activation de la séquence de menu Vue.Visual. Dans ces deux cas, le compsant formulaire est ouvert et l'éditeur de formulaire remplace l'éditeur de projet dans la zone de travail centrale. L'option de menu Projet est replacée par deux nouvelles options de menu Formulaire et élément graphique et, si la barre d'outils à gauche de l'écran est active, son contenu change pour montrer les différents outils de fabrication d'éléments graphiques (éléments graphiques) offerts par l'éditeur de formulaires.

Ce serait une bonne chose d'activer la barre d'outils située à gauche et la barre de palette située à droite si cela n'est pas déjà fait.

La barre d'outils d'éditeur de formulaires présente deux colonnes d'outils d'éléments graphiques et d'icônes; en bas de la boîte d'outils une seule icône centrée indique le type de l'élément graphique ou outil sélectionné.

Un type d'élément graphique ou outil peut être sélectionné pour l'utilisation en cliquant sur l'icône correspondant avec le bouton de la souris. Cet icône est ensuite affiché au bas de la boîte d'outils en tant que sélection courante.

Essayez ces différents types, en choisissant le type d'élément graphique ou outil courant, pour se familiariser avec cette opération. On peut dire que la sélection courante est indépendante de l'affichage de la boîte d'outils. La sélection courante est préservée et restorée lors de l'affichage ou non de la barre d'outils.

Avant de commencer à utiliser ces outils, ce serait une bonne chose d'essayer de cliquer sur les couleurs offertes dans la barre de palette, d'abord avec le bouton gauche de la souris puis avec le bouton droit de la souris.

Pour en comprendre l'impact, il est nécessaire que la barre d'outils soit visible.

En regardant précisément l'icône Texte à la quatrième ligne de la première colonne, vous pouvez noter qu'un clic avec le bouton gauche de la souris affecte la couleur sélectionnée à l'avant plan ou les caractères du texte. Un clic avec le bouton droit affecte la couleur de fond du texte.

Essayez plusieurs fois et notez la façon dont se modifient différentes icônes d'éléments graphiques lorsque sont sélectionnées les couleurs de fond et d'avant-plan dans la barre de palette.

Il peut également être noté que certaines icônes d'éléments graphiques ne répondent pas aux événements de modification de couleurs car elles sont standardisées en utilisant les couleurs réservées de l'interface utilisateur graphique dans l'intervalle de 16 à 31. Cette étendue des couleurs diffère en fonction du fichier de palette courante, définie par la variable d'environnement GIGO, la variable de configuration PALETTE ou le champ de palette par défaut de l'éditeur de projet Camera.

Un dernier point concernant les couleurs : la première entrée dans la palette, de valeur 0, vue comme du noir, est en fait utilisée pour représenter la transparence. Cela peut être vérifié par un clic bouton droit sur la couleur suivi de l'examen de l'icône Elément Texte de la barre d'outils.

On peut noter que la zone de travail centrale est couverte par une grille qui, quand elle est active, est utilisée pour le positionnement et les calculs de dimension de tous les éléments graphiques.Les dimensions et positions des éléments graphiques seront effectivement vérrouillées aux intersections du quadrillage si la grille est active.

les dimensions de la grille et son état peuvent être modifiées, entre autre, en activant la séquence de menu Options.Paramètres faisant apparaître la boîte de dialogue Paramètres Généraux .

Essayez de changer les valeurs des champs Largeur de Grille et Hauteur de grille avant d'appuyer sur le bouton Valider et d'observer la grille de l'éditeur de formulaires.

Avant de continuer, restaurez les largeur et hauteur de grille à 10, comme valeurs par défaut .

La même boîte de dialogue permet également de préciser des valeurs par défaut pour les propriétés d'éléments graphiques Fonte, Alignement, Gras, Souligné et Ombré. Ces valeurs par défaut, ainsi que les couleurs sélectionnées via la barre de palette, comme décrit précédemment, sont utilisées au fur et à mesure lors de la création de nouveaux éléments graphiques qui s'ajoutent au formulaire.

Continuons maintenant afin d'utiliser les différents types d'éléments graphiques et outils , la barre d'outils et la palette devant être actives :

  1. Choisir l'icône de type Fenêtre sur la première colonne et la sixième ligne.

  2. Assurez-vous que cela devienne le type d'élément graphique ou outil courant en bas de la boîte d'outils.

  3. Déplacez le curseur de la souris sur la cellule la plus en haut à gauche de la grille.

  4. Appuyez et conservez enfoncé le bouton gauche de la souris.

  5. Déplacez le curseur de la souris vers la droite et en bas, aussi loin que vous le voulez , tout en restant dans la zone de travail.

  6. Pendant que vous maintenez enfoncé le bouton de la souris et que vous déplacez le curseur, vous pouvez noter :
    • le rectangle vert représentant la position et les dimensions du nouvel élément graphique
    • ainsi que les coordonnées de la position et les dimensions dans la zone de message la plus à gauche dans la barre de statut.

  7. Relachez le bouton de la souris et vous pouvez voir un composant fenêtre à la place du rectangle vert.

  8. La zone de message centrale de la barre de statut affiche maintenant les caractéristiques, classe, nom, position et dimensions du composant de formulaire ou élément graphique courant.

  9. La zone de message gauche de la barre de status affiche le nom du formulaire et ses dimensions totales, dans ce cas celles du seul composant fenêtre.

Vous pouvez appuyer sur la touche Suppr du clavier pour supprimer le composant et répéter l'opération autant de fois que vous le sentez nécéssaire.

Quand vous vous sentez à l'aise avec la procédure précédente, en gardant votre dernier composant fenêtre créé, passez à la suite :

  1. Choisissez l'icône de l'élément graphique Bouton poussoir à la cinquième ligne de la seconde colonne

  2. Déplacez le curseur souris quelque part à l'intérieur du composant fenêtre

  3. Appuyez et maintenez enfoncé le bouton gauche de la souris

  4. Déplacez le curseur souris, en le laissant à l'intérieur du composant fenêtre , pour définir les dimensions du composant

  5. Relachez le bouton de la souris pour ajouter le composant Bouton Poussoir au formulaire.

Vous devriez maintenant voir le composant bouton poussoir visible dans votre composant fenêtre et nous pouvons générer notre premier programme visuel avec sing.

L'activation de la séquence de menu Fichier.Production, depuis l'éditeur de formulaires, ouvre la boîte de dialogue Production Abal, permettant de modifier les différentes options de production. Pour le moment, activez simplement le bouton Valider pour confirmer l'opération de production et retourner à la vue de l'éditeur de formulaires.

Comme résultat de l'opération précédente, les trois opérations suivantes se produisent

  1. Le source abal représentant le formulaire élémentaire, comprenant une fenêtre et un bouton est écrit dans le fichier new.as précisé par la propriété Nom de Fichier de la boîte de dialogue Production Abal.

  2. Le source abal est traduit en utilisant l'utilitaire de traduction défini dans la propriété de configuration.

  3. le modèle de formulaire est enregistré dans le fichier de ressource xml du projet, sauf si cela a été désactivé dans la boîte de dialogue Paramètres Généraux.

Vous pouvez regarder le source abal généré par Sing en activant la séquence de menu Vue.Texte L'éditeur texte interne ouvre le contenu du fichier source dans la zone centrale. Pour être capable de comprendre l'architecture et les détails du source généré, vous pourrez trouver utile de consulter les documents Production de Source Abal et la Bibliothèque Visual Abal.

Pour pouvez maintenant exécuter le programme nouvellement créé en cliquant sur le bouton d'utilitaire Exécution Normale ou l'utilitaire Exécution en mode Debug de la barre de menu en haut de l'écran.

Pour quitter le programme visual, vous devez appuyer sur la touche Echap du clavier puisque jusqu'à présent nous n'avons pas défini le comportement des composants fenêtre et bouton.

Vous êtes maintenant revenu à l'éditeur de formulaires, après l'éxécution du programme :

  1. Déplacez le curseur souris dans le composant fenêtre mais pas sur le composant bouton

  2. Appuyez et maintenez enfoncé le bouton gauche de la souris

  3. Notez que le composant fenêtre devient le composant actif courant comme affiché dans la zone centrale de la boîte de statut.

  4. Déplacez le curseur de la souris dans la zone centrale du bureau

  5. Notez les points suivants:
    • Le composant fenêtre est déplacé en fonction du mouvement de la souris
    • Le composant bouton est également déplacé en conservant sa position dans la fenêtre.
    • L'information sur la position courante du composant fenêtre est reporté dans la zone de message centrale de la barre de statut.

  6. Relachez le bouton gauche de la souris pour établir la nouvelle position de la fenêtre et de son bouton.

Essayez maintenant la chose suivante

  1. Déplacez le curseur de la souris sur le composant bouton

  2. Appuyez et maintenez enfoncé le bouton gauche de la souris

  3. Notez que le composant bouton poussoir devient maintenant le composant actif courant au vu de l'information affichée dans la zone centrale de la barre de statut.

  4. Déplacez le curseur de la souris

  5. Notez le mouvement du composant bouton par rapport au mouvement de la souris mai également que la position de la fenêtre n'est pas affectée

  6. Notez l'information de la zone centrale de la barre de statut concernant la position courante du composant bouton poussoir.

  7. Relâchez le bouton de la souris pour établir la nouvelle position du composant bouton poussoir.

Suite aux précédentes opérations il faut comprendre que seul le composant fenêtre, parmi tous les autres types d'éléments graphiques, se comporte comme un Ensemble de Composants alors que le composant bouton poussoir, ainsi que les autres types d'éléments graphiques, se comportent comme des composants singuliers.

Maintenant nous pouvons continuer pour vous montrer comment redimensionner les composants éléments graphiques.

  1. Déplacez la souris sur le composant à redimensionner

  2. Appuyez et maintenez enfoncé le bouton droit de la souris.

  3. Le composant devient le composant actif dans la zone central de la barre de statut.

  4. Déplacez la souris jusqu'à ce que le composant atteigne les dimensions requises.

  5. Les dimensions du composant sont affichées dans la zone centrale de la barre de statut.

  6. Relachez le bouton droit de la souris pour établir les dimensions finales.

Vous pouvez maintenant répéter les deux opérations de repositionnement et de redimensionnement pour vos deux composants de formulaire, jusqu'à ce que vous vous sentiez complètement l'aise avec ces techniques car cela sera fréquemment nécessaire dans la suite de votre travail avec l'éditeur de formulaire.
