Accept Production Properties

This visual form will be presented to allow inspection and modifica tion of application model production parameters prior to the activa tion of the corresponding production operation.

ModelName
The MODELNAME edit field presents the name of the application model for inspection.

TargetFile
The PRODUCTION filename edit field presents the production target filename for modification or inspection. The value of this edit field will be used as required by the project production nature control to provide the name of the file to which global production will be sent.

TargetType
This control allows selection of the project production nature. A value may be taken from the following list :
none
no production is performed other than that required for the individual model components.
program
the project model is a program source, all text items will be generated as inclusions.
module
the project model is a module source, all text items will be generated as inclusions.
overlay
the project model is an overlay source, all text items will be generated as inclusions.
library
the project model is a class library source, all text items will be generated as inclusions.
package
the project model is a product package and will generate a makefile describing the various components of the product.

All application model types other than none will perform the corresponding post production operation as required by the specific model type.

ProdLang
This check box allows selection of the production language associated with the project production model nature defined by the preceeding select control.

Document
The DOCUMENT check box when checked allows the production of an HTML document containing all Application model Images and documentary text from the sub-components. This will be generated to the Document Filename target if one has been specified.

DocumentFile
This edit field allows a full path and filename to be specified for the top level HTML document to be produced in response to the activation of document production by the corresponding check box.

Makefile
When this control has been checked the production mechanism will generate a description of the entire project to a standard makefile.

MakefileName
This edit field allows the full path and filename of the file to which the project make file will be generated if the corresponding check control has been activated.

Visualise
The VERBOSE check box controls the visualisation of the various application components as they are presented for effective production.

Automatic
The AUTOMATIC check box when checked alleviates the need to validate the various stages involved in the production of the target and its documentation. When unchecked each stage will present the corresponding visual frame for validation or cancellation.

Force Build
The FORCE BUILD check box when checked performs production of all partial products in a systematic manner. When unchecked only the entities which are out of date will be produced.

By pressing the ACCEPT button any changes made to the various parameters will take effect.

Edit
By pressing on the EDIT button the document source will be presented for inspection via the Standard Text Editor mechanisms.

Html
By pressing on this button you will gain access to the Html Document Generation parameters for inspection and eventual modification.

View
By pressing on the VIEW button the system navigator will be activated allowing the finalised version of the production document to be consulted. Under UseIT NETSCAPE will be activated.

By pressing the CANCEL button any modifications performed will be discarded.
