  1. General
    This page presents the various parameters controlling the configuration operations and runtime animation scripts.

    • FILENAME
      This edit field displays and offers for modification the configuration localisation identifier or name.

      The value of this field will be used as the name of the configuration file, postfixed with .cfg, by the configuration file operations, load, save and edit. It will also be used in the construction of the distinguished name, prefixed with config, by ldap configuration operations ldap load and ldap save.

    • ABALHOST
      This edit field defines the source and target host machine to be used for subsequent ldap configuration operations.

      It receives its initial value at startup from the environment variable of the same name, which when legal and not null, activates ldap configuration loading.

    • IDENTITY
      This field simply provides an identification string to be displayed in the tree view mechanisms simply for configuration recognition purposes.

    • REPLAY
      This edit field allows an animation replay script filename to be defined for the automation of a visual applications. For more details concerning animation scripts please refer to the document to this effect.

    • RECORD
      This edit field allows an animation recording script filename to be defined for use by visual applications. All keyboard and mouse activity will be recorded to the specified file during the execution of the application.

  2. Abal
    This page presents the default values of the collection of parameters controlling ABAL source translation, object module linking, program execution and debugging.

    • ATR
      This field defines the default full path and file name of the ABAL translator to be used for the translation of all standard ABAL syntax production streams.

    • ATR_DEFINE
      In conjuction with the preceeding field this field accepts the various command line definitions required for the translation of standard ABAL syntax production streams. They should be presented as a space or comma separated list of definition names or definition name=value pairs. The list will be parsed to generate the standard abal definition command line syntax.

    • ATR_INCLUDE
      This edit field allows a space or comma separated list of include paths to be defined for the localisation of incluse files during the translation operation. The list will be parsed to generate the standard abal command line syntax. The defined order of the various paths will be respected.

    • ATR_OPTIONS
      This field allows default values to be specified for the command line switchs

      Owhen activated this will inhibit optimisation of the generated tcode.
      Mwhen activated causes generation of symbolic information for debug.
      VActivates verbose messages displayed to the system text screen.
      TActivate error tracing using the internal error tracing mechanisms.

    • ALD
      This will provides the full path and file name of the module linker to be used for of programs, resulting from standard ABAL syntax translation, and requiring to be linked with object modules for the production of their executable result.

    • ALD_OPTIONS
      This field allows the default values of the various linker option switchs to be defined.

      Mthis option activates inclusion of modular symbolic information in the final program.
      Vcontrols display of informational messages to the system text screen.
      Scontrols display of source phase tracing messages to the system text screen.
      Tcontrols display of target phase tracing messages to the system text screen.

    • OTR
      As for the ATR field above, this field allows the full path and file name of the translator to be used for the translation of ABAL++ class syntax production streams.

    • OTR_DEFINE
      This fields allows for the specification of control definitions for the translation of class object production streams.

    • OTR_INCLUDE
      In an identical fashion to the standard ABAL INCLUDE above, this field allows the list of search paths to be specified for included files. It will also be used as a base for the library search path for object libraries.

    • OTR_OPTIONS
      This field allows definition of the default values of the command line switchs defined above concerning the standard ABAL syntax translation values.

    • OLD
      This edit field allows the full path and filename of the object module linker to be defined for programs resulting from the class object production streams.

    • OLD_OPTIONS
      This edit field allows the object linker command line switch default values to be defined.

    • EXA
      This edit field allows the full path and filename of the ABAL runtime or executer to be defined.

    • EXADB
      This edit field accepts the definition of the fill path and filename of the ABAL debug utility to be used for the debug of visual programs.

  3. Images
    This page provides the collection of parameters to be used by the visual subsystem for the retrieval of images, definition of colours and other widget oriented options.

    • IMAGEPATH
      This edit field allows definition of the space or comma separated list of search paths to be used for the localisation of the various image files required by the visual tools and visual applications. The image file search mechanisms will respect the order defined by this value.

    • ICONPATH
      This edit field defines the single icon image search path as used by the visual tools for the retrieval of the various icon image files.

    • PALETTE
      This edit field defines the full path and file name of the colour map palette to be loaded and used for the definition of the 256 colours of the graphical subsystem. These palettes are a PROLOGUE standard derivation of the standard GIF palette. The first 16 colours are reserved for use by the standard ABAL and CICO colour instructions, the 16 colours which follow are reserved for use by the visual user interface mechanisms affording a high degree of colour conformity accross the range of visual widgets. The remaining colours are the standand values of the internationl JPEG/GIF 256 colour map palette.

    • SKIN
      This edit field allows the full path and file name of an image to be used for surface tiling to be defined. Surface tiling implies the horizontal and vertical repetition of a smaller image unit in order to cover the entire surface with a consistant texture.

    • TRIGGER_STYLE
      This select control allows the default trigger value to de selected. Trigger components, tab pages, buttons check boxs and radio buttons, may specify an ALT key trigger by prefixing the corresponding character of their label by the ampersand character. The trigger component may specify the trigger character display format by setting either of the BOLD or UNDERLINE atributs of its option mask parameter. When neither is specified the default value provided by this parameter will take effect.

    • TRIGGER_COLOUR
      This edit field provides the colour to be used for default trigger display when the trigger format is set to COLOUR.

  4. Fonts
    This page presents the collection of default values used by the visual font management subsystem. Values defined here include the font search path and the various font file names of the fonts loaded into the font slots of the font manager mechanisms.

    • FONTPATH
      This edit field defines the search path to be used for all font load and save operations for filenames which do not specify an explisite path component.

    • FONT1
      This edit field allows the font file name of the fixed pitch standard edit font to be specified. This font is used in all edit fields of the visual design tool and should be at most 10 pixels wide by 16 pixels. high.

    • FONT2
      This edit field provides the name of the font file loaded into the second font slot of the visual design tool font management mechanisms. This font is used by the visual design tool for all control widgets, menu options, push buttons, tab frames, check boxs and radio buttons. It should be a proportional spaced font of varying width somewhere between 16 and 24 pixels in height.

    • FONT3
      This edit field controls the name of the font loaded into the third font slot. This should be a proportional spaced font slightly larger than that present in font slot two.

    • FONT4
      This edit defines the filename of the font loaded into slot 4, again proportionaly spaced and slight larger than that in slot 3.

    • FONT5
      The file name of the font loaded into slot 5 is defined here. Again it should be preoprtionaly spaced and slightly larger than the font present in font slot 4.

    • FONT6
      This edit field defines the name of the font to be loaded into the font slot 6. This is to be largest of the title fonts, being larger than the font loaded into slot 5 and as for the others proportional spaced.

    • FONT7
      This edit field defines the name of the font loaded into font slot 7. This font has no reserved usage and is initially a fixed pitch font of 10 pixels wide and 24 pixels high. It provides a larger standard edit font for certain applications.

    • FONT8
      The file to be loaded into font slot 8 is defined here. This is reserved as the small texts font, used on window buttons for the exit and help characters and in other tool icon buttons of the visual design tool. It should be as small as possible, by default a font of 4 pixels wide by 8 pixels high is loaded.

    • FONT9
      This edit field defines the name of the font loaded into font slot 9, a variation of the font in slot 7, it has no reserved application.

    • FONT10
      This edit field defines the font loaded into font slot 10. This font is currently undefined and unused.

  5. Documents
    This page presents the default values of the various parameters involved in the Html Docuement Production mechanisms of the visual design tool.

    • HTML_TITLE
      This edit field provides the value of the message to be used in the Html Docuement TITLE and page HEADER constructions.

    • HTML_LOGO
      This edit field defines the name of an image file to be used as the document header logo in the document header and table of contents.

    • HTML_COPYRIGHT
      This edit field provides a text to be used in the generation of the document copyright section, or legal notice, at the foot of each document.

    • SCREEN_CSS
      This edit field provides the url or file name of the style sheet to be referenced by the screen style instructions of the generated document. Style sheets are used in document production allowing full externalisation of all aspects of final document style. For more information to this effect please consult the corresponding documents concerning Cascading Style Sheets at the world wide web consortium site.

    • PRINT_CSS
      This edit field defines the style sheet to be used in the style expressions relating to html document printing by the eventual document navigator.

    • NAVIGATOR
      This edit field provides the name and full path name of a system utility to be used for Html Document Navigation and Inspection.

      Under Useit this would be mozilla, and under Twin Server the Twinternet Navigator nv.

    • DOCPATH
      This edit field defines the search path name prefix for all Html docement activation operations using the above defined navigator. This may equally well be an url of the type http://server/path/ or a simple local directory path prefixed by the standard file service prefix file:///.

  6. Printing
    This page presents the default values of the various parameters used in the production of application component graphical documents.

    • PRINTER
      This edit field defines the full path and filename of the ABAL Overlay program to be as the print controller in the production of application document components.

      The sources of the default value program printer.at are supplied in the home abal3 class directory in the form of the class file gprinter.as. This is an ABAL++ Object Overlay Program that may be used equally well from standard ABAL syntax as from ABAL++ Class Object production scenarios.

    • PAPERTYPE
      This select control allows selection of the paper size from the following table of standard values, showing also the total number of dots in both horizontal and vertical directions.

      PAPER751001503006001200
      A4
      A3
      A2
      A1

    • LAYOUT
      This select control allows choice of the paper layout or orientation.

      • Portrait being the standard vertical layout
      • Landscape the alternative horizontal layout.

      The underlying printer mechanisms will perform the various image transformations as required by the value of this parameter.

    • PAGEFEED
      This select control allows choice of page feed behaviour. This is printer and environment dependant and may chosen from amongst the following

      • None, no page feed is generated , the printer will eject the page when complete.
      • Before,a page feed is generated prior to starting page output.
      • After,a page feed is required to eject the page when a page of data has been sent.

      Consultation of the printer manual may be necessary for the choice of this parameter.

    • RESOLUTION
      This select control provides the default value to be used for application document components for their graphical print resolution.

      This value is in dots per inch.

      It defines the overall print density and consequently controls the total number of dots available per line and page.

      This number of dots will be used in conjunction with the actual application component width and height, and the paper size parameter, to calculate the most appropriate number of dots or bits to be used to represent each source pixel in the final printed image.

      The greater the resolution or density, the greater the acuracy of the final grey shade or colour scale conversion.

  7. Editor
    This page presents the default and current values of the various parameters used by the version of the ABAL text editor, aed, internal to the visual design tool.

    • AED
      This edit field defines the full path and filename of the editor to use for all source and html text editing operations.

      If the first character of the string is a ! or exclamation mark, the internal ABAL editor will be used for editing purposes.

      When an external editor is defined for use, under Useit, it will be opened in a new system window and launched with an independant system process.

    • AED_FONT
      This select control allows selection of the font to be used by the internal ABAL text editor. The use of fixed pitch fonts is strongly recomended.

    • AED.STATE
      This check control when active inhibits the display of the editor configuration pannel prior to startup of a text editor operation. When unset the control pannel will be presented for configuration parameter confirmation.

    • AED.SYNTAX
      This check control,when set, activates the syntaxical colouration of the source during the internal editor operation. When unset, syntaxical colouration will be inhibited.

    • AED_NORMAL
      This select control allows selection of the colour to be used by the internal ABAL text editor to represent normal text, user defined naming tokens for variables, procedures and functions.

    • AED_QUOTE
      This select control defines the colour to be used to represent quoted string literal values.

    • AED_KEYWORD
      This select control defines the colour to be used to represent language keywords.

      The internal ABAL text editor may operate with syntaxical colouration for the following source languages :

      • ABAL and ABAL++
      • C and C++
      • HTML

    • AED_COMMENT
      This select control defines the value to be used to represent language dependant commentary blocks.

    • AED_PUNCTUATE
      This select control defines the colourto be used to represent standard punctuation characters,arithmetical and comparision operators.

    • Accept
      By clicking on this button the current values of all configuration parameters will be accepted and made available to the visual desing tool replacing the current active values prior to termination of the configuration dialog box activity.

    • SaveFile
      When this push button is actioned the collection of configuration parameters will be accepted and written out to the filename identified by the configuration parameter FILENAME of the first page.

      Once the file save has completed, control will be returned to the configuration dialog box.

    • SaveLdap
      By clicking on this button the current values of the configuration parameters will be written to the LDAP Directory on the network host identified by the IP Address taken from thez configuration parameter ABALHOST. The value of the field FILENAME will be used, prefixed by config , to identify the collection of configuration in the directory.

    • LoadLdap
      This push button provides the complementary action to the SaveLdap push button. with the ABALHOST parameter being used to identity the LDAP host and the FILENAME parameter to identify the configuration collection.

    • EditFile
      When this push button is pressed, the file identified by the configuration parameter FILENAME will be open using the standard text editor mechanism for inspection and eventual modification.

    • Delete
      When this push button is pressed, the configuration collection identified by the FILENAME parameter will be deleted from the LDAP directory on the host defined by the ABALHOST parameter.

    • LoadFile
      Activation of this push button will cause the file identified by the configuration paramater FILENAME to loading into the configuration manager and made available for inspection and modification by the configuration dialog box.

    • Cancel
      By clicking on this push button any modifications to the configuration parameter set will be discarded prior to leaving the configuration dialog box.

    • AMAKE_DEFINE
      This edit field defines the full path and filename of the ABAL amake utility.

      This tool will be activated in response to clicking on the Execution button of the menu tool bar for any source file item named makefile or with the .mf or .mak file extension.

    • AMAKE_LIST
      This edit field defines the full path and filename of the ABAL amake utility.

      This tool will be activated in response to clicking on the Execution button of the menu tool bar for any source file item named makefile or with the .mf or .mak file extension.
