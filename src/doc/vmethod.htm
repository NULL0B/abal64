accept forms method

This dialog box is presented allowing the inspection and modification of a forms method.These methods are defined at forms level and may be accessed from any other the widget level or other forms level methods.

Accept
The ACCEPT button allows the modifications performed to the forms method properties to be kept and applied to the parent form.

Full Screen
By clicking on this button, the full screen source editor will be activated for the edition of the forms method source.

Cancel
The CANCEL button allows modifications performed tn the forms method properties to be abandoned and discarded.

  1. Method
    This tab page presents the collection of forms method parameters for inspection and eventual modification.

    • Public
      This check control applies to object source generation only, and when active indicates that the forms method may be acccessed from outside of the forms class. Otherwise the method will be private.

    • Pointer
      This check control when activate indicates that the associated return value will be a pointer to the type indicated by the select control ReturnType.

    • MethodType
      This select control allows definition of the return type of the forms method taken from the table presented below :

      TypeDescription
      VoidMethods of this type will not return values.
      Byte8 bit integer return type.
      Word16 bit integer return type.
      Long32 bit integer return type.
      NumericBcd Numeric return type
      StringString return type
      ClassClass return type.

    • MethodClass
      This edit field allows the name of the class to bespecified for class return types

    • Inline
      This collection of radio controls applies to object source generation only.

      The inline option implies that the application method source will be inlined where invoked rather than packaged in a procedural function body.

    • Function
      This radio option indicates procedural function packaging of the forms method application code.

    • Virtual
      When this radio option is selected the function method will contribute to the virtual interface of its parent form.

    • Overload
      When this radio option is selected the forms method function will overload a virtual function of the parent class or one of its baseclass components.

    • MethodName
      This edit field defines the formal name of the forms method.

    • Signature
      This edit field allows definition of the parameter variables for the reception of any eventual parameters values required to be passed to the forms method.

  2. Body
    This tab page presents the application source code associated with and defined for the method.

    • MethodText
      This edit field allows the application source code to be edited using the internal source editor.
