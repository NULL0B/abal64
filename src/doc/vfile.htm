database editor

This dialog frame gives access to the collection of data or file widgets defined and used by the various forms of the project.The various definitions are collected from all data widgets defining data or file structural aspects during project and form loading. The resulting class reference is stored back to the widget allowing retrieval of the now centralised information. In this way all data and file widget information is unified to be made available to all of the visual forms comprising the project.

The data base information will be stored to the project ressource file, in XML format, unless an individual database ressource name is provided by the project description, in which case the data and file widget descriptions will be stored to the indicated external database ressource file, and may consequently be shared amongst different projects.

  1. List
    This tab page presents the list of all data base items.

    • FileList
      The item list displays the various file or data entities and a summary of their overall basic type. By clicking on a selection with the mouse it will be made available in the file data page for inpsection and modification.

    • ScrollBar
      The scroll bar control may be used to scroll through data base information exceeding the visual page size.

  2. File
    This tab page presents the individual information pertaining to one data or file item of the database collection.

    • ClassName
      This edit field provides the public class name of the data description, by which it will be made available to the data widgets of the client forms.

    • Instance
      This edit field allows an advisory or standard instance name by which the data will be accessed.

    • Filename
      This edit field provides the advisory file name in which the data records will usually be stored. This and the instance name may be over-ridden as required by the corresponding fields of the client data widget.

    • FileType
      This selection control allows the basic data or file type to be specified.

    • IndexType
      This selection provides the optional primary index characteristics required for sequential indexed and multiple indexed file types.

    • IndexSize
      This edit field allows the overall size in bytes of the primary index, of sequential indexed and multiple indexed files, to be specified. This field is of no importance to other data types.

    • -
      The data record member count may be decremented by clicking on this button.

    • NbIndex
      This edit field displays the current number of members comprising the primary index.

    • +
      The primary index member count may be incremented by clicking on this button.

    • VD=
      The data record of sequentail index and multiple indexed files may be declared to be of variable length by means of this check control.

    • RecordSize
      This edit field allows the overall size in bytes of the data record to be specified for sequential, sequential indexed and multiple indexed files.

      It will provide the storage handle, a number between 31 and 63, for virtual memory files.

    • -
      The primary index member count may be decremented by clicking on this button.

    • NbData
      This edit field displays the current number of members comprising the data record.

    • +
      The data record member count may be incremented by clicking on this button.

    • Accept
      This button may be activated allowing the current contents selected by the data file mechanisms, to be returned as the corresponding class identifier to the client form data widget.

    • IndexMembers
      This table will be displayed when the primary index member count is greater than zero indicating the presence of primary index member information. The various columns of valid member items may be modified as required. Clicking in the type field will make a selection control appear for the prompted selection of the member data type.

    • DataMembers
      This table will be displayed when the data record member count is greater than zero indicating the presence of data record member information. As for the index member information, the various columns of valid member items may be modified as required. Clicking in the type field will make a selection control appear for the prompted selection of the member data type.

  3. Mask
    This tab page gives access to the data mask generation controls for this data file.
