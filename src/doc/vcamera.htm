This visual form will be presented to allow inspection and modifica tion of the various application model parameters. These include the camera view parameters and the various default settings used by the application model entities.

AtX
The POSITION X,Y and Z edit fields give access to the coordinates o f the current application camera position for inspection and modif ication.

ToX
The FOCUS X and Y edit fields offer the application camera focus fo r inspection and modification. The Camera Focus implies the deplac ment from the 0,0 center position of the application view.

Direction
The DIRECTION edit field shows the current camera direction taken from the following : FORWARD, BACKWARD,UP,DOWN LEFT or RIGHT. Whilst in the Application Viewer the direction may be changed by simply pressing the first letter of the required direction name.

Configuration
The TIMER edit field allows a value to be specified for the automation of the application camera which when used in conjunct ion with the STEP field value allows automised deplacement of the camera view.

DataBase
The TIMER edit field allows a value to be specified for the automation of the application camera which when used in conjunct ion with the STEP field value allows automised deplacement of the camera view.

Step
Thre STEP edit field allows the camera movement step value to be modified for automatic displacment.

Accept
By pressing the ACCEPT button any changes made to the various parameters will take effect.

Cancel
By pressing the CANCEL button any modifications performed will be discarded.

StepX
The DISTANCE X,Y and Z edit fields provide access to the current camera movement step values when a camera movement key is action ed. The camera action keys are the PAGEUP and PAGEDOWN keys which move in the current direction which may be toggle by the s pace bar amongst the following values : X,Y or Z direction.

ImageFormat
This select control allows the image format to be specified. Currently the only formats that may be produced are BMP and GIF. The later producing mush smaller files but may require longer image save times.

AutoSize
The AUTOSIZE check box provides the default value to be used when creating new application model entities.

Save Forms
The SAVEIMAGES check box and SAVEFORMS check box provide the defaul t beahvious to be used by the application model save operation. The y may me overidden by hand in the corresponding visual form.

ImageName
The IMAGE edit field allows the full path and filename of an image to used as the application model background over which the 3d ca mera entities will be projected.

PaletteName
This edit field allows the file name of a palette file to be specified. This palette will be loaded and used for all subsequent project models.

ImageType
This combo selection field allows a value to be chosen from a widget specific list of options.

The semantics of the options vary depending on the actual class of the widget as shown in the following table.

WIDGETBehaviour
WINDOWThe alignment value applies to the window itself within the parent graphical output screen.
TEXT
None
no alignment is applied and the text will be displayed with automatic line wrap within the zone defined by the position and dimension values. This alignment value is not intended for use with proportional fonts.
Right,Left,Center
These values will cause the text to be displayed aligned in a single line centered in a vertical direction within the zone.
Justify
The text will be space padded displayed such that it touches both left and right edges of its zone
FRAMESFor the various frame components, namely INSET,OUTSET,GROOVE andRIDGE, The alignment value applies to the payload which , when provided, will be displayed as a textual title for the frame. The alignment value of none inhibits title display.
IMAGEThe following image options are available.
None
no alignment is applied to the image.
Right
The image will be right aligned in the widget zone.
Left
The image will be left aligned in the widget zone.
Center
The image will be centered within the widget zone.
Tile
This option causes the images, smaller than their widget zone, to be tiled or repeated, to fill the entire zone.
Makefit
causes images to be stretched or squashed to fit the entire widget zone.
Bestfit
causes images to stretched and squashed respecting the image proportions. The result may stil be smaller than the widget zone in which case it will be centered.
SCROLLBARThis option controls the composition of the scroll bar. In all cases, when the zone width is found to be greater than the zone height a horizontal bar will be displayed. Otherwise, when the width is less than the height, a vertical bar will be displayed.
None
The scroll bar trough will occupy the entire widget zone. No butons will be displayed.
Right
A down or right button will be displayed at the appropriate extremity. The scroll bar trough will occupy the remaining zone.
Left
A up or left button will be displayed at the appropriate extremity. The scroll bar trough will occupy the remaining zone.
Both
Both up and down or left and right buttons will be displayed at each end of the bar. The scroll bar trough will occupy the remaining zone.
Slider
No buttons will be displayed and the slider bar will be displayed as a rounded slider in a grooved trough, occupying the entire widget zone.
OTHERNo semantic is currently defined for other widget types.
