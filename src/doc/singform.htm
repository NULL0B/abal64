Forms Editor

The visual forms and document modelling tool of the ABAL 3 development environment allows the development of application programs on a visual basis.

Application forms and document models are described in terms of their visual components here after refered to as widgets. The application may be considered an object scenario in which the various widget component actors interoperate to achieve the applications global purpose or objectives.

The application model may be generated to produce one of the following four application packages :

Program
Program components provide applications with their underlying base scenarios in much the same way as was performed by by segments in bal and abal. Program packages are responsible for configuration of the application environment in terms of the graphical resolution, fonts, palettes and paths to mention but a few.
Overlay
Overlay components provide complete external service packages for use in application scenarios through dynamic linkage mechanisms. Packages of this type are intended primarily for use with ABAL 3 programs but provide also for the needs of other application developement clients.
Module
Module components provide incomplete external services designed for static linking to program or overlay compoents. Again, these packages are intended for use by ABAL 3 Programs and Overlays but also provide for the needs of third party application architectures.
Inclusion
These components are seldom used in homogenous ABAL 3 applications, but are made available for use during the transition period for hetrogenous application design.

For the construction of these forms and document models the visual forms editor provides the widgets, tools and menu operations described in the remainder of this document.

FormTools
The Visual Forms editor is the main application design tool of the ABAL 3 development environment.

It offers a collection of widget tools for the elaboration of the visual forms or document model scenario.

A widget tool may be selected by clicking over its image on the tool bar at the left of the desktop area.

The icon of the currently selected widget tool can be seen at the foot of the forms editor tool bar.

The behaviour of the various widgets will now be described in terms of the following basic categories

  1. Decorative Widgets

    Widgets which fall into this category are visible and contribute to overall appearance of the forms or document model. Widgets outside of this category are invisible and consequently do not contribute to the models appearance.

  2. Declarative Widgets

    Widgets of this category instance variables providing for data input / output and storage.

  3. Active Widgets

    These widgets are capable of performing various actions such as the collection, separation or presentation of other widgets, atributs or data.

Each of the various widget tools will be examined in detail, particularily in terms of the three above defined categories.

Palette
The Palette bar displays all colours of the currently loaded palette and allows colour selection by a simple click on the required colour. A click with the left button will set the current foreground colour whilst a click with the right button will set the current background colour. Selection of a colour for the project model background is possible in this way, The effect will be visible at one unless a full screen background image has already been affected to the project modeler background.

Event
Selection of this menu option gives access to the menu of widget component methods and documentation.

OnCreate
Selection of this menu option causes the corresponding method text of the current forms widget to be made available via the standard text editor for consultation and eventual modification. The instructions provided in this way will be invoked during the construction phase of the parent form thus allowing the widget the opertunity to perform all actions necessary for its correct initialisation.

These methods are called in the order that the widgets are instanced or declared in the parent form.

Forward referencing of widgets and their as yet un-initialised parameters from construction method text is not at all recomended and may give unpredictable results.

OnRemove
Selection of this menu option causes the corresponding method text of the current forms widget to be made available via the standard text editor for consultation and eventual modification. The instructions declared here will be invoked during the destruction phase of the parent forms model allowing the widget to perform all actions required for the completion of its operation.

The destruction methods will be invoked in the inverse of the widget instance order to ensure perfect complementary action to the construction phase.

Again forward, or rather reverse, referencing of widgets and their properties from destruction method text is extremly bad practice.

OnShow
Selection of this menu option causes the corresponding method text of the current forms widget to be made available via the standard text editor for consultation and eventual modification.

ItemDocument
Selection of this menu option gives access to the Html Editor allowing inspection modification of the documentation associated with the current active widget of the current forms model.

Documentation and Information provided in this way will be collected together along with the information provided at the forms compoent level, to build the forms documentation during the project model production operation.

Triggers
Selection of this menu option allows the calculation and affectation of widget trigger ALT keycodes for all valid active widget types. The Hotkey Triggers check control of the Forms Parameters dialog box must have been activated to allow this operation to proceed.

To be considered valid for inclusion in the calculation of widget triggers, a control widget must have a valid user provided name and not have the Lock atribut set.

This operation is performed as follows :

  • All locked widget triggers are collected for the form and used to initialise the trigger distribution map.

  • All unlocked widget triggers are reset to zero, inactive.

  • New and unique widget triggers are calculated and affected where possible.

  • When no more widget triggers are available the standard error/ warning dialog box will be displayed for user notification.

Widgets may be triggered by pressing the and holding the ALT key then pressing the underlined trigger key.
This causes a special trigger event type to be returned through the mouse and keyboard event channel.
The following collection of event bit fields will be set :

  • MIMO KEY : indicates a keyboard type event.
  • MIMO DOWN : this and the following are raised to indicate a click type event.
  • MIMO UP
  • MIMO ALT : formalises the trigger type event.

When received through the keyboard channel, widget trigger events will return the standard MOUSE EVENT code, 256, indicating that the event information is to be read through the event channel functions.

  • Event(1) returns the event type bit field as decribed above.
  • Event(2) returns the alt code code value formed by combining the keycode value with the hexadecimal flag /800. For example the character A whose keycode is /41 would return the value of /841 in this way.
  • Values returned by the other event functions are undefined.

Interface
Selection of this menu option causes all default interface widget types, that have not been locked, to contribute to the interface of the forms model component.

This is of great importance for MODULE type packages which may need to receive certain values as parameters for their correct operation.

The following widget types are chosen by default to contribute to this interface

Widget TypeParameters
EDITa stringvalue$,stringlength% pair for each edit widget
CHECKone value% per check widget
RADIOone value% per named group of radio widgets
SWITCHone value% per switch widget
SELECTone value% per select widget

Individual widgets may be removed from the interface collection by unchecking the corresponding control in the Widget Properties dialog box of the concerned widget.

Locked widgets will not be included in the interface collection created by this menu option.

Widget
Selection of this menu option gives access to the following popup menu of widget specific functions and operations. In all cases the operation may equally apply to either the single current widget, or, in the case of the current widget being a selection or cut frame, the collection of widgets completly encompassed by its position and dimensions.

Font
Selection of this menu option causes the value of the global parameter Options Parameters Font of the Global Parameters dialog box to be used as the text font for the current widget or selection of widgets.

Align
Selection of this menu option causes the value of the global parameter Options Parameters Align of the Global Parameters dialog box to be used as the alignement or option field of the current widget or selection of widgets.

Bold
Selection of this menu option toggles the value of the Bold parameter of the current widget or collection of widgets.

Shadow
Selection of this menu option toggles the value of the Shadow parameter of the current widget or collection of widgets.

Underline
Selection of this menu option toggles the value of the Underline parameter of the current widget or collection of widgets.

Copy
Selection of this menu option causes the current widget or collection of widgets to be copied to the forms editor clipboard.

Paste
Selection of this menu option causes the widget or widgets currently copied to the forms editor clipboard to be pasted into the form after the current widget. All method texts and properties will be duplicated and widgets with automatic names will be renamed during this operation. The clipboard will still contain copies of the source widgets after the operation is complete.

Lock
Activation of this menu option causes the property value Lock of the current widget or selection of widgets to be set or locked. Widgets locked in this way may not be moved or redimensions, nor may their widget trigger value be recalculated using the global menu option Triggers.

Unlock
Activation of this menu option causes the property value Lock of the current widget or selection of widgets to be reset or unlocked.

Automatic
Activation of this menu option causes the property value of the Show method of the current widget or selection of widgets to be set to Automatic. This is the default value for this parameter and it implies that the code required for the display of the widget will be generated to the global forms display method OnShow.

Conditional
Activation of this menu option causes the property value of the Show method of the current widget or selection of widgets to be set to Conditional. In this case the code required for the display of the widget will not be generated to the global forms display method OnShow. Instead it will be generated when and where invoked by other co-operating widgets from their scenario integration methods.

Foreground
Selection of this menu option causes the current foreground colour of the global palette mechanisms to be used for the current widget or collection of widgets.

Background
Selection of this menu option causes the current background colour of the global palette mechanisms to be used for the current widget or collection of widgets.
