Introduction

This document presents a preliminary and non contractual view of certain aspects of the phase 2 of the project Abal 3.

This phase of the project pertains to the addition of functionallity allowing access to SQL servers, ORACLE, MY SQL etc.

This document outlines eventual technical specifications of these new SQL extensions.

The underlying file access mechanisms, of the ABAL runtime module EXA, offer a collection of functions which may be used as a common language base from which to start.

New keywords will be required to handle new concepts available with SQL.

Abal Syntax

The following instructions will need to be modified to provide acces to these new SQL functions.

ASSIGN

ASSIGN=<handle>,<name>,SQL[<error>]

READ

READ=<handle>[<error>,<variable>]

WRITE

WRITE=<handle>[<error>,<variable>]

The following instructions will be added to the standard ABAL syntax to offer acces to these new SQL functions.

SELECT

SELECT=<handle>,<expression>[<error>]

PREPARE

PREPARE=<handle>,<expression>[<error>]

FETCH

FETCH=<handle>,<expression>[<error>]

EXEC

EXEC=<handle>,<expression>[<error>]

The following instructions will be added to accomodate and normalise the features previously made available by the MODEXT interface.

KEYTYPE

KEYTYPE=<handle>,<expression>[<error>]

LINKNAME

LINKNAME=<handle>,<expression>[<error>]

FILEUSER

FILEUSER=<handle>,<expression>[<error>]
