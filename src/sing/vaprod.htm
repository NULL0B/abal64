accept abal production

This visual form will be presented as a result of the selection of the generic menu option File Production whilst the Visual Forms Editor is active for an ABAL forms model.

It allows the various parameters and values involved in Abal Source Production and compilation to be inspected and modified. Abal Source Production will be performed for forms models where the language type has been set to ABAL or ABAL++.

  1. Production
    This page presents the general production parameters.

    • FileName
      The ABAL++ Production Filename field identifies the full path and filename of the target to which the production of source will be generated.

    • Identity
      This usage of this edit field value depends on both the production language and the production packaging.

      For ABAL++ production ir provides the class name.

      For the production of an ABAL module it provides the stub or root name of the module interface and support procedures.

      For ABAL PROGRAM production packages it will be used un the generation of the PROGRAM statement.

      In all cases it will be used as the title and identification of the model in html document production.

    • Packaging
      This control allows selection of the ABAL production packaging model from the following list

      ModelDescription
      ProgramProduction packages of this type are intended to fulfil the requirements of application root scenarios to which modules may be linked, sources included and to which overlays may be attached.
      ModuleThese production packages are suitable for the implementation of resident application services for use from various application platforms.Modular production includes the generation of the corresponding external reference description file.
      OverlayThese production packages are intended for the implementation of transient application services again to be shared amongst applications.
      IncludeThis packaging is intended for the production of re-usable visual components in the form of a class file to be included by other packaging types for the resolution of visual complex data types.

    • IsAutoTrad
      When this control is checked secondary production phases will be performed without prior user confirmation. Secondary production involves source translation and eventual object linking as described by the various production parameters.

    • Accept
      By pressing the ACCEPT button the Abal Production operation will be performed to the indicated target file. Secondary production will also be scheduled as required by the production parameters.

    • Cancel
      By pressing this button the Abal Production operation will be abandoned.

    • Auto6307

    • Dimensions
      This control allows selection of the target graphics resolutions required for the production of program packaging. The optimal value will be calculated prior to presentation of this dialog frame. It may be increased as needed to encompass the requirements of secondary components.

    • UseAtr
      This button provides direct access to the secondary production or translation phase required for all ABAL production packing models except inclusions. The filename indicated in the Target filename field will be submited for translation via the Abal Translator dialog frame.

    • Link
      This button provides a short cut to the final phase of secondary production for ABAL program and overlay packaging models for which module linking is required.

    • UseEdit
      By pressing the EDIT button the filename indicated in the Target filename field will be opened for inspection using the standard Text Editor.

    • Select
      The Select button allows access to the file selection dialog frame allowing prompted selection of the production target filename.

  2. Inclusion
    This page allows the names of the various files required to be included to be specified. The names may be specified in any order and will be sorted in terms of their extension to be included as follows :

    EXTENSIONDescription
    DEFLibrary definition files will be included prior to the generation of their packaging statement.
    CLSFiles bearing this extension will be included prior to the generation of the program module or overlay packaging.
    DCLFiles bearing this extension will be included after the production of the model data section.
    EXTFiles bearing this extension will be included prior to the generation of the internal procedures
    OTHER All other file extensions will the be included prior to the generation of forward, extern and procedure statements.

    • Dependance
      This edit field allows the list of include files to be specified as a space or comma separated list of file names.

  3. Modules
    This page allows the list of modules to which the program or overlay is to be linked to be specified. Modules for which a corresponding visual model file is detected will give rise to the generation of the corresponding external description inclusion.

    Models for which modules are declared will be scheduled for the secondary production module linkage phase.

    • Linkage
      This edit field allow specification of a space or comma separated list of abal module filenames including their extension.

  4. PragmaPage
    This page provides a collection of translation pragmas and definitions allowing fine control over the production and runtime context.

    • UseDiezeMem
      This control may be checked to permit generation of the corresponding #MEM statement.

    • UseDiezeStack
      This control may be checked to allow generation of the corresponding #STACK statement.

    • UseDiezeFiles
      This control may be check to allow generation of the corresponding #FILES statement.

    • UseDiezeHeap
      This control may be checked to allow generation of the corresponding #HEAP statement.

    • DiezeMem
      This edit field provides the value to be used for the #MEM statement which determines the amount or temporary storage memory to be preallocated by the ABAL RunTime for use in calculations and procedure nesting.

      It should be at least twice as large as the largest temporary string operation.

    • DiezeStack
      This edit field provides the value to be used for the #STACK statement which determines the depth of gosub,segment and procedure nesting. As a general rule segments and gosubs use one entry eacj whilst procedures may require between 12 and 16 items depending on the nature of the call.

    • DiezeFiles
      This edit field provides the value to be used for the #FILES statement which determines the total number of files that may be assigned at any one time by the ABAL runtime. The number of files which may actually be opened is not controlled by this value being rather a function of the host operating system.

    • DiezeHeap
      This edit field provides the value to be used for the #HEAP statement. This construction is of little use in modern operating system environments but may improve performance in memory critical situations. The value may be calculated as the total amount of nested local memory.

    • RespectCase
      This control may be check forcing the translator to ignore differences in case when translating variabel, constant and procedure names. When this control is unchecked then names of different case will refer to different objects.

    • OptimiseTables
      This control when checked signals that the translator is to perform variable table optimisation for the elimination of unused variable identifiers. This option is not to be used for modules.

    • UseTokenSize
      This control may be checked to allow generation of the corresponding token size statement.

    • UseLabelSize
      This control may be check allowing generation of the corresponding label size statement.

    • TokenSize
      This edit field provides the value to be used for the token size pragma statement. By default tokens are 16 bytes long. They may be increased or reduced to any reasonable value between 8 and 64 bytes.

    • LabelSize
      This edit field provides the value for the label size pragma statement. As for the token size pragma this may take any reasonable value between 8 and 64 bytes.
